# EventSure 데모 영상 스크립트

## 개요

- **프로젝트명**: EventSure
- **한 줄 소개**: Mantle Network 기반 탈중앙화 항공편 지연 보험 플랫폼
- **핵심 가치**: 해석 불필요. 청구 불필요. 신뢰 불필요. (No interpretation. No claims. No trust needed.)
- **예상 영상 길이**: 5-7분

## PART 4: 에피소드 탐색하기

### 화면

Explorer 페이지 (`/explorer`)

### 행동

1. 헤더에서 "Explorer" 클릭
2. 에피소드 리스트 화면 노출
3. 필터 버튼 (All, Open, Locked, Resolved) 클릭해보기
4. Open 상태의 에피소드 카드 클릭

### 스크립트

> "이제 실제로 에피소드를 탐색해 보겠습니다."
>
> "Explorer 페이지에서 현재 진행 중인 모든 보험 에피소드를 확인할 수 있습니다."
>
> "상태별로 필터링할 수 있습니다. Open은 참여 가능한 에피소드, Locked는 비행 중인 에피소드, Resolved는 정산 완료된 에피소드입니다."
>
> "각 카드에는 항공편 정보, 출발 시간, 트리거 조건, 보험료와 최대 보상금이 표시됩니다."
>
> "참여 가능한 에피소드를 클릭해 보겠습니다."

## PART 5: 에피소드 참여 Flow

### 화면

Explorer 상세 페이지 (3단계 스텝퍼)

### 행동

#### Step 1: Rules (규칙 확인)

1. 스텝퍼에서 "1. Rules" 활성화된 상태
2. Episode Rules 섹션 천천히 스크롤
3. 보장 이벤트, 오라클, 재무 조건, 풀 로직 설명
4. 체크박스 클릭하여 동의
5. "Join Episode" 버튼 클릭

### 스크립트

> "에피소드 참여는 3단계로 진행됩니다."
>
> **[Step 1: Rules]**
> "첫 번째 단계는 규칙 확인입니다."
>
> "보장 이벤트를 확인합니다. 어떤 항공편의, 어떤 조건이 트리거되는지 명확히 표시됩니다."
>
> "예를 들어, 이 에피소드는 '도착 지연 3시간 이상'이 트리거 조건입니다."
>
> "오라클 정보도 확인할 수 있습니다. FlightAware API를 데이터 소스로 사용하고, 도착 직후 결과가 결정됩니다."
>
> "재무 조건도 투명합니다. 보험료, 최대 보상금, 추가 기여금 여부가 모두 표시됩니다."
>
> "풀 로직을 읽어보면, 동질적 위험 풀이고 잉여금은 조합원에게 비례하여 반환된다고 명시되어 있습니다."
>
> "이 규칙들에 동의한다면 체크박스를 클릭하고 Join Episode 버튼을 누릅니다."

#### Step 2: Join (풀 가입)

1. 스텝퍼에서 "2. Join" 활성화
2. 보험료, 최대 손실, 최대 보상금 확인
3. 카운트다운 타이머 표시
4. 경고 메시지 확인
5. 지갑 연결 (RainbowKit 모달)
6. "Confirm & Join" 버튼 클릭
7. 지갑에서 트랜잭션 승인

### 스크립트

> **[Step 2: Join]**
> "두 번째 단계는 실제 참여입니다."
>
> "최종 확인 화면에서 보험료와 최대 손실, 최대 보상금을 다시 한번 확인합니다."
>
> "풀 마감까지 남은 시간이 카운트다운됩니다."
>
> "중요한 경고문이 있습니다. '참여하면 이 에피소드의 리스크 풀 멤버가 됩니다. 참여 후에는 환불이 불가능합니다.'"
>
> "지갑이 연결되어 있지 않다면 먼저 연결해야 합니다."
>
> **[지갑 연결 시]**
> "RainbowKit을 통해 MetaMask, WalletConnect 등 다양한 지갑을 지원합니다."
>
> "지갑이 연결되면 Confirm & Join 버튼을 클릭합니다."
>
> "지갑에서 트랜잭션을 승인하면 참여가 완료됩니다."

#### Step 3: Dashboard (모니터링)

1. 스텝퍼에서 "3. Monitor" 활성화
2. 실시간 비행 상태 확인
3. 현재 지연 시간, 트리거까지 진행률 바
4. "실시간 데이터는 참고용" 안내 문구

### 스크립트

> **[Step 3: Dashboard]**
> "세 번째 단계는 모니터링입니다."
>
> "참여한 에피소드의 실시간 상태를 확인할 수 있습니다."
>
> "비행 상태, 현재 지연 시간, 트리거 조건까지의 진행률이 시각적으로 표시됩니다."
>
> "하단의 안내문처럼, 실시간 데이터는 참고용일 뿐이고 최종 결과는 지정된 시간에 오라클에 의해 결정됩니다."
>
> "이것이 신뢰 없이 작동하는 보험의 핵심입니다."

## 데모 체크리스트

### 사전 준비

- [ ] Mantle Sepolia 테스트넷에 테스트 MNT 보유
- [ ] MetaMask 또는 지원 지갑 설치 및 연결
- [ ] Open 상태의 에피소드가 최소 1개 이상 존재
- [ ] 브라우저 개발자 도구 닫기
- [ ] 화면 해상도 1920x1080 권장

### 데모 중 주의사항

- 지갑 트랜잭션 승인 시 Gas Fee 화면이 보일 수 있음
- 네트워크 상태에 따라 트랜잭션 확인까지 시간이 걸릴 수 있음
- 이미 참여한 에피소드에서는 "Already Joined" 상태가 표시됨

## 기술 스택 언급용

영상 끝이나 설명 중 언급할 수 있는 기술 스택:

- **블록체인**: Mantle Network (Ethereum L2)
- **스마트 컨트랙트**: Solidity
- **프론트엔드**: React 19, TypeScript, Vite
- **Web3**: wagmi, viem, RainbowKit
- **스타일링**: Emotion (CSS-in-JS), Framer Motion
- **오라클**: FlightAware API (항공편 데이터)
- **다국어 지원**: i18next (한국어, 영어)
